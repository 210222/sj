场景 1：Python for 循环 — 代码高亮 + 执行流程图
for 循环是怎么执行的？
看这段代码： 
fruits = ['apple', 'banana', 'cherry']
for fruit in fruits:
    print(f'I like {fruit}')
print('Done!')
执行流程是这样的：
📊 Mermaid — 代码执行流程
flowchart TD
    A["创建列表 ['apple','banana','cherry']"] --> B["取第 1 个: 'apple'"]
    B --> C["执行 print('I like apple')"]
    C --> D{"还有下一个?"}
    D -->|"是"| E["取第 2 个: 'banana'"]
    E --> F["执行 print('I like banana')"]
    F --> D
    D -->|"是"| G["取第 3 个: 'cherry'"]
    G --> H["执行 print('I like cherry')"]
    H --> D
    D -->|"否"| I["执行 print('Done!')"]
    I --> J["✅ 程序结束"]
循环就是"取一个→处理→取下一个"的重复。range(3) 等价于 [0,1,2]。
场景 2：数据结构对比 — 列表 vs 链表（内存布局图）
列表和链表的区别是什么？
列表在内存里是连续排列的——就像电影院里的连号座位：
📊 Mermaid — 列表 (连续内存)
graph LR
    A["a 0x100"] --> B["b 0x108"] --> C["c 0x110"] --> D["d 0x118"]
链表在内存里是散落的——每个节点存了"下一个在哪"的地址：
📊 Mermaid — 链表 (分散 + 指针)
graph LR
    A["a ·→"] -->|"指针"| B["c ·→"]
    B -->|"指针"| C["b ·→"]
    C -->|"指针"| D["d ·→"]
时间复杂度对比： 关键区别：列表快速随机访问(O(1))但插入慢(O(n))。链表插入快(O(1))但找第 k 个元素慢(O(n))。
场景 3：递归可视化 — 斐波那契调用栈（Mermaid 时序图）
递归是怎么一层层调用的？
斐波那契数列的递归实现： 
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

print(fib(4))  # 输出: 3
fib(4) 的调用树：
📊 Mermaid — 递归调用树
graph TD
    A["fib(4)"] --> B["fib(3)"]
    A --> C["fib(2)"]
    B --> D["fib(2)"]
    B --> E["fib(1)=1"]
    D --> F["fib(1)=1"]
    D --> G["fib(0)=0"]
    C --> H["fib(1)=1"]
    C --> I["fib(0)=0"]
递归的本质是"函数调用自己，直到触底反弹"——fib(1) 和 fib(0) 是底，直接返回。然后一层层往上算。

场景 4：面向对象继承 — Mermaid 类图 + 代码
Python 的类和继承是怎样的？
class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return f'{self.name}: Woof!'

class Cat(Animal):
    def speak(self):
        return f'{self.name}: Meow!'
