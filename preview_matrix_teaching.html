矩阵初学者最佳教学图策略

核心原则: KaTeX 展示"矩阵长什么样" + Mermaid 展示"概念间什么关系" — 两者互补，不重复

❌ 错误示范：抽象概念分类树 初学者看不懂

这种图只罗列名词，没有具体数字实例。初学者看完只知道"有这些分类"，完全不知道矩阵到底长什么样。

❌ 坏的 Mermaid 图

节点只有一个名词，边没有说明，没有具体例子

✅ 好的组合策略

KaTeX 先展示具体实例，Mermaid 再展示概念关系

KaTeX 给了心理图像 + Mermaid 给了知识结构 = 完整理解

✅ 策略一：具体实例优先 最核心

先展示带具体数字的最小实例，让学生"看到"概念长什么样，再给抽象定义。
对矩阵来说：statement 中的 KaTeX 公式就是最好的"图"。

标注每个元素的位置名称——让学生知道怎么描述"第2行第3列的数字是6"

用具体数字展示变换——转置就是把"第1行变成第1列"

✅ 策略二：展示变换过程（矩阵乘法）关键

教操作/运算时，展示"操作前 → 操作过程 → 操作后"。具体数字走一遍，比任何抽象描述都有效。

1
维度检查
左矩阵列数(2) = 右矩阵行数(2) ✓
结果矩阵: 2×2
2
行 × 列 点积
c₁₁ = 第1行·第1列
= 1×5 + 2×7 = 19
3
逐个计算
c₁₂ = 1×6 + 2×8 = 22
c₂₁ = 3×5 + 4×7 = 43
c₂₂ = 3×6 + 4×8 = 50
4
得到结果
C = [[19,22],[43,50]]
2×3 × 3×2 = 2×2

✅ 策略三：并排对比区分（加法 vs 乘法）防混淆

当两个概念容易混淆时，同一组输入数据，并排展示两个操作的结果，差异一目了然。

教学要点：加法和乘法完全不同——①规则不同(对应位置 vs 行列点积) ②结果尺寸可能不同(同尺寸 vs m×p) ③是否有交换律(A+B=B+A 但 AB≠BA)

✅ 策略四：Mermaid 展示矩阵知识结构

当学生对矩阵有了基本认知后，用 Mermaid 展示完整的知识体系——图展示"和什么有关+分几类"，KaTeX 展示"是什么+怎么变"。

flowchart LR subgraph 基础概念 A["矩阵: m×n个数的矩形阵列"] B["元素: aᵢⱼ = 第i行第j列的数"] end subgraph 基本运算 C["加法: 对应位置相加"] D["数乘: 每个元素乘常数"] E["转置: 行列互换 A→Aᵀ"] F["乘法: 行×列 点积求和"] end subgraph 特殊矩阵 G["方阵: m=n"] H["单位矩阵I: 对角线全1"] I["逆矩阵A⁻¹: AA⁻¹=I"] end subgraph 应用 J["解线性方程组"] K["图像变换"] L["数据分析"] end A --> C A --> D A --> E A --> F G --> H G --> I C --> J F --> K F --> L E --> K
subgraph 分组让结构清晰：基础→运算→特殊→应用，逐层递进。每组内的节点用不同颜色区分。 不用在一轮里全讲完——每轮讲一个 subgraph 就够了。
