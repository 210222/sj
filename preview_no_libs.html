零外部依赖方案预览

方案 A：纯 SVG（零依赖，浏览器内置，AI 输出 XML 文本）
冒泡排序 [5,3,8,1] 每一步怎么变的？
一步步看：
🎨 纯 SVG — 无需任何 JS 库
AI 输出方式：纯 XML 文本（<svg>...</svg>）
优点：零 npm 依赖，浏览器原生支持，轻量，无限灵活
缺点：无交互（不能拖拽）。复杂图需要 AI 输出大量坐标
方案 B：Canvas API（浏览器内置，AI 输出 JS 绘图指令）
sin(x) 在 [0, 2π] 的图像是什么样？
🖌️ Canvas API — 无需任何库
AI 输出方式：JS 绘图指令（ctx.moveTo / ctx.lineTo / ctx.fillText）
优点：零依赖，高性能（GPU 加速），可做动画
缺点：AI 需要输出 JS 代码——比 SVG 更难生成，出错风险高

方案 E：后端渲染（Python matplotlib → PNG 图片）

AI 输出方式：Python 代码字符串
优点：能力无上限——matplotlib 能画任何图。3D、统计、地图
缺点：需要后端代码执行 + 图片存储 + 安全沙箱。复杂度最高

推荐
零依赖方案（SVG/CSS/Unicode）可以作为轻量补充——当 npm 库因网络问题不可用时自动降级。
但作为主力方案，npm 库（Mermaid+KaTeX+Desmos+Prism）的 AI 生成难度和图形质量明显更优。

最终建议：主力 = npm 4 库。降级备用 = 纯 SVG + CSS + Unicode。
后端 matplotlib 作为远期方案（Phase 63+），不放在 Phase 62。
